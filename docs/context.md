# Project Context

Gym Engine is a graphics-first learning system built around interactive visual experiments.

The original seed idea was a mathematics course inspired by classic computer graphics books that made higher math feel exploratory, visual, and rewarding.

That course became the Prueitt Mathematics Gym.

The broader idea is now to build several parallel Gyms that share a common experimentation engine.

---

## Learning Philosophy

The project is based on the belief that difficult concepts are easier to understand when they are:

- implemented directly
- visualized immediately
- explored through parameter changes
- revisited across multiple contexts

This repository intentionally favors experimentation over passive study.

The learner should see:
- curves form
- fields emerge
- terrain evolve
- agents move
- systems stabilize or destabilize
- patterns self-organize

---

## Why Gyms

A Gym trains a conceptual muscle through repeated exposure.

But the Gym model should stay flexible.

Important rule:

A Gym is a collection, not a strict locked sequence.

Concepts are allowed to overlap.

Examples:
- optimization belongs to Math and AI
- stochastic processes belong to Math, Simulation, and Emergence
- search belongs to Algorithms and AI
- recursion can appear in curves, procedural growth, and L-systems

---

## Why Experiments Matter More Than Chapters

The repository should grow by adding experiments.

Experiments are the atomic units.

Each experiment should:
- represent one clear visual system
- teach one or more concepts
- be runnable independently
- expose parameters
- carry tags

This allows new interests to be added at any time without rewriting completed material.

For example, a future interest like plant growth should be added as a new experiment, not as a reason to restructure the entire repository.

---

## Curiosity-Driven Expansion

The learner will continue to discover new things worth studying:
- plant growth
- procedural ecosystems
- neural decision boundaries
- galaxy formation
- ant colony optimization
- Fourier analysis
- swarm intelligence
- fluid systems

The project should welcome this.

New ideas should enter through a lightweight framework:
- concept
- experiment
- tags
- optional Gym links

---

## Long-Term Vision

This repository should become a personal visual research playground.

It should support:
- learning difficult technical concepts
- building reusable visualization tools
- testing emergent systems
- practicing simulation and AI ideas
- creating portfolio-quality experiments

Over time it should accumulate a library of small, visually satisfying, concept-rich experiments that reinforce one another.
